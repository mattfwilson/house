<objective>
Define the SQLite schema as Drizzle tables (`profiles`, `scenarios`), wire the better-sqlite3 + Drizzle connection + programmatic migrator (`db.ts`), then GENERATE and commit the real migration SQL and prove it applies to a live database. Money columns are TEXT (canonical decimal strings), the scenario `snapshot` is a TEXT blob (the canonicalJson source of truth, D-05), and a unique index enforces scenario-name-uniqueness-within-profile (D-11).

Purpose: This is the persistence substrate every repository in 06-05 builds on. The [BLOCKING] migration task is the load-bearing proof that the schema is materialized in a real DB — Drizzle's row types come from the schema module, so a passing `tsc -b` is a FALSE-POSITIVE for "the table exists." Only a migrate-then-insert/select round-trip proves it.
Output: `schema.ts`, `db.ts`, committed `drizzle/*.sql` migration(s), and a migration test that round-trips a real row through a fresh `:memory:` DB.
</objective>
