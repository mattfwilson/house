<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SQLite TEXT blob → in-memory EngineInput | A stored/forged snapshot becomes a trusted EngineInput only through the Zod re-parse on load |
| DB row columns → Profile | Profile columns are re-validated through parseProfile (DB is not blindly trusted) |
| in-memory values → SQL statement | Drizzle parameterizes all writes/reads |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-12 | Tampering | scenario blob load | mitigate | LOAD re-parses every leaf through parseAssumptionSet/parseScenarioInputs/parseHousehold + calendarDate — a forged/corrupt blob throws, never silently computed (ASVS V5). No `as` cast. Asserted by grep gate + a forged-blob test. |
| T-06-13 | Tampering (data integrity) | money persistence | mitigate | Money round-trips as canonical decimal TEXT; profile load re-validates via parseProfile (decStr). The byte-identity round-trip proves no float re-entry corrupts the snapshot. |
| T-06-14 | Tampering | SQL queries | mitigate | Drizzle parameterizes all insert/select/delete; no string-concatenated SQL (ASVS V5). |
| T-06-15 | Integrity | reproducibility regression | mitigate | save→reload→canonicalJson byte-identity asserted with plain `toBe` (not toMatchSnapshot, which auto-re-blesses) — preserves the Phase-1 PROF-04 contract under SQLite. |
</threat_model>
