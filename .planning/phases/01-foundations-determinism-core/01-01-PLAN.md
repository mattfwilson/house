<objective>
Bootstrap the npm-workspaces monorepo and the `packages/core` skeleton, and stand up the build-time enforcement that makes CORE-01 (no framework in core) and the lint half of CORE-03 (determinism) CI failures rather than conventions. This is the Wave 0 scaffolding every later plan depends on: workspaces, tsconfigs (core has NO DOM lib / NO JSX), ESLint 10 flat config with the boundary + determinism rule blocks, Vitest 4 `projects` wiring, dependency installs, and the negative fixtures that prove the guards actually fail the build.

Purpose: The value of this phase is the *enforcement*, not the lines of code. A boundary that can be bypassed is worthless — this plan proves a framework import, a DOM global, and ambient nondeterminism each break the build.
Output: Root config files, `packages/core` skeleton, installed deps, and committed negative-test fixtures + a boundary test asserting lint/typecheck reject violations.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm registry → local install | Third-party package code enters the project at install time. |
| developer source → CI build | Lint/tsc/test gates are the boundary that rejects framework imports and ambient nondeterminism. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-SC | Tampering | npm installs (decimal.js, zod, eslint plugins, vitest, tsc) | mitigate | All packages CLAUDE.md-prescribed + registry-verified; blocking human-verify checkpoint (Task 1) before first install; commit `package-lock.json`; recommend `npm ci` in CI. |
| T-01-01 | Tampering | Framework import sneaking into core (CORE-01 silently unmet) | mitigate | `boundaries/external` deny-by-default (covers dynamic import) + `no-restricted-imports` + core tsconfig no-DOM; negative fixture `boundary.test.ts` proves lint rejects `import 'react'`. |
| T-01-02 | Repudiation (of correctness) | Ambient nondeterminism (Date.now/Math.random/env) entering core | mitigate | `no-restricted-syntax` + `no-restricted-globals`/`-properties` lint block (runtime guard half added in Plan 02). |
| T-01-03 | Information Disclosure | Lint config accidentally scopes determinism rules to whole repo, masking apps later | accept | Rules are `files`-scoped to `packages/core/**`; apps tier is deferred and will get its own block — low risk, no current apps dir. |
</threat_model>
