<objective>
Implement assumptions-as-first-class-data (ASMP-01) and the deterministic input contract. Build the nested `AssumptionSet` (namespaced by domain), its versioned Zod schema (discriminated union on integer `schemaVersion`, decimal-string serialization), the `migrate(old) -> current` path, a versioned defaults module (pure seed data), and the `EngineInput` type that threads `asOf` + `assumptions` + scenario inputs as one immutable object — the exact shape a snapshot captures (D-11). This makes "nothing hardcoded" enforceable and gives Plan 04's golden harness a stable input unit.

Purpose: Every downstream module reads its own assumption slice instead of hardcoding tunables; versioning + Zod validation make stored snapshots reproducible and forgery-resistant before persistence exists.
Output: `assumptions/` module (schema, types, defaults, migrate) + `engine/engine-input.ts`, with unit tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| serialized snapshot / assumptions JSON → in-memory AssumptionSet | Untrusted-or-corrupt data is validated by Zod safeParse before use (defense against a corrupt fixture, not an attacker). |
| numeric literals → assumption values | Only canonical decimal strings cross; JS floats are rejected. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | Malformed/forged AssumptionSet snapshot deserialized | mitigate | Zod `safeParse` on every parse; `discriminatedUnion` rejects unknown `schemaVersion`; `migrate()` gates on version. |
| T-03-02 | Tampering (of precision) | Float re-entering at the serialization boundary (defeats D-06) | mitigate | `decStr` regex validator instead of `z.number()`; round-trip test asserts no `0.035000…` artifact. |
| T-03-03 | Tampering | Prototype pollution via JSON.parse of a snapshot | mitigate | Validate with Zod immediately after parse; never spread an unvalidated parsed object into config. |
| T-03-04 | Repudiation (of correctness) | Hardcoded tunable bypassing AssumptionSet (ASMP-01 silently unmet) | mitigate | All tunables live in the nested schema + defaults module; downstream reads slices — enforced by review + the canary reading from AssumptionSet in Plan 04. |
</threat_model>
