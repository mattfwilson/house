<objective>
Assemble the **top-level `computeTco`** aggregator (TCO-06): take a frozen widened `EngineInput`, derive loan basics (loan = price x (1 - downPaymentPct), LTV — D-14), resolve the town's FY-stamped mill rate, compose the Plan 02 + Plan 03 calculators, and return a closed **monthly + annualized breakdown** of every line (P+I, tax, insurance, maintenance, HOA, PMI, amortized closing) with all dollars as `Money` and the resolved rate+FY captured for the snapshot. Lock the no-bare-number result shape with a `tco.type-test.ts`, and publish the public barrel.

Purpose: This is the single composition point the Affordability engine (Phase 3 reuses PITI+HOA+PMI for DTI) and the rent-vs-buy model (Plan 05) consume. The type-test makes "a dollar field is never a bare number" a build-time guarantee (mirrors `money.type-test.ts`). It depends on both Wave 2 plans (amortization/PMI and tax/carrying/closing).

Output: New `tco/tco.ts`, `tco/tco.test.ts`, `tco/tco.type-test.ts`, and an updated public barrel `index.ts`.
</objective>
